Add the data foundation for token analytics: database schema extension, shared types, token estimation service, cost service extension, and heartbeat integration.

Purpose: Without this data layer, no analytics UI can display token breakdowns, cache hit rates, or context window utilization. This plan creates the server-side infrastructure that TOKN-01, TOKN-02, and TOKN-03 depend on.

Output: DB migration adding cachedInputTokens to cost_events, shared types for token breakdown and model context limits, token estimation service, extended cost service with cache/token aggregation, heartbeat integration that stores breakdown in usageJson and cachedInputTokens in cost_events.
</objective>
