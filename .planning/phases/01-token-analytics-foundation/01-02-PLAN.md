    2. **Create `server/src/services/claude-usage-streaming.ts`:**

       ```typescript
       interface ParsedUsage {
         inputTokens?: number;
         outputTokens?: number;
         cachedInputTokens?: number;
       }

       /** Parse a single JSON line from Claude stream-json for usage data. */
       export function parseClaudeUsageFromLine(line: string): ParsedUsage | null;

       /** Parse a chunk (potentially multiple lines) for usage data. Returns all found usage updates. */
       export function parseClaudeUsageFromChunk(chunk: string): ParsedUsage[];

       interface UsageTrackerOptions {
         emitIntervalMs?: number; // default 2000
         onEmit: (usage: { inputTokens: number; outputTokens: number; cachedInputTokens: number }) => void;
       }

       interface UsageTracker {
         processChunk: (chunk: string) => void;
         flush: () => void;
         getCurrent: () => { inputTokens: number; outputTokens: number; cachedInputTokens: number };
       }

       /** Create a debounced usage tracker that parses chunks and emits at intervals. */
       export function createUsageTracker(opts: UsageTrackerOptions): UsageTracker;
       ```

       Implementation details:
       - `parseClaudeUsageFromLine`: Try JSON.parse each trimmed non-empty line. Look for usage in:
         - `event.usage` (top-level usage object)
         - `event.message?.usage` (nested in message_start events)
         - `event.result?.usage` (nested in result events)
         Extract `input_tokens`, `output_tokens`, `cache_read_input_tokens` (note: snake_case from API).
       - `parseClaudeUsageFromChunk`: Split chunk by `\n`, parse each line, collect non-null results.
       - `createUsageTracker`: Maintains cumulative usage state. On each processChunk, parses and takes max of current vs parsed (monotonic increase). Uses timestamp check for debouncing: only calls onEmit if >= emitIntervalMs since last emit AND values changed. `flush()` emits immediately if values > 0.
