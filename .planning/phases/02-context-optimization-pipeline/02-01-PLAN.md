<objective>
Build the context optimization pipeline: a processor chain that transforms agent execution context before it reaches the adapter. Includes task-type resolution from issue labels, structured brief serialization, context deduplication, and cache-optimized prompt ordering.

Purpose: This is the core optimization engine for Phase 2. All four processors work together to reduce token consumption by compacting context, routing by task type, removing redundancy, and structuring prompts for cache hits.

Output: `server/src/context-pipeline/` module with types, runner, and 4 processors, plus `packages/shared/src/types/task-types.ts` shared type. All with comprehensive tests.

  <behavior>
    Task-type resolver (TOPT-03):
    - Label "bug" resolves to "bug_fix"
    - Label "feature" resolves to "feature"
    - Label "review" or "code review" resolves to "review"
    - Label "refactor" or "refactoring" resolves to "refactor"
    - Unknown labels fall through to auto-detection from issue content
    - Auto-detect: "fix crash" -> bug_fix, "add new feature" -> feature, "review PR" -> review, "refactor code" -> refactor
    - Null issue and no labels -> taskType stays null (generic template used)
    - First matching label wins (multiple labels)
    - Case-insensitive label matching

    Context serializer (TOPT-01):
    - Produces StructuredBrief from issue data
    - Description truncated to ~2K chars for non-bug tasks, ~4K for bug_fix (per Pitfall 4)
    - Comments truncated to ~500 chars each, only last 3 non-triggering comments
    - Triggering comment preserved in full
    - Strips orchestration metadata keys from context (timestamps, internal IDs)
    - Replaces context with serialized brief representation
    - Measures compression ratio (original vs compressed token estimate)
    - Returns unchanged context if issue is null

    Deduplicator (TOPT-04):
    - Removes context fields that duplicate information already in the structured brief (e.g., if contextSnapshot has issueTitle and the brief has it)
    - Strips empty string fields and null values from context
    - Preserves essential non-issue context fields (workspace, projectId, issueId)

    Prompt reorderer (TOPT-05):
    - Prompt template structure is deterministic (same agent config -> same template prefix)
    - Returns ctx unchanged if no reordering needed (structural, not per-run)
    - Documents the 4-layer ordering expectation for downstream integration
  </behavior>
