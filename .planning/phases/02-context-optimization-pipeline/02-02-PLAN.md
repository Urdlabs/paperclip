       - Add optional `onBudgetWarning?: (usage: { inputTokens: number; outputTokens: number; cachedInputTokens: number }, budget: { maxTokens: number; windDownThreshold: number }) => void`

    2. Inside `createUsageTracker`, add budget enforcement:
       - Add `let windDownTriggered = false` state
       - In `processChunk`, after updating cumulative state and detecting `changed`:
         ```
         if (changed && opts.budget?.maxTokens && !windDownTriggered) {
           const totalUsed = inputTokens + outputTokens;
           const threshold = opts.budget.maxTokens * (opts.budget.windDownThreshold ?? 0.9);
           if (totalUsed >= threshold) {
             windDownTriggered = true;
             opts.onBudgetWarning?.({ inputTokens, outputTokens, cachedInputTokens }, {
               maxTokens: opts.budget.maxTokens,
               windDownThreshold: opts.budget.windDownThreshold ?? 0.9,
             });
           }
         }
         ```

       - Test: onBudgetWarning not called when budget is null
       - Test: onBudgetWarning not called when usage is below threshold
       - Test: isWindDownTriggered returns true after warning fires
       - Test: inputTokens + outputTokens used as total (not just input)
