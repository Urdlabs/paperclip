  <name>Task 1: Trace tree transformation utility with tests</name>
  <files>ui/src/lib/trace-utils.ts, ui/src/lib/trace-utils.test.ts</files>
  <behavior>
    - buildTraceTree([]) returns empty array
    - buildTraceTree with a single init entry returns one root node of type "init"
    - buildTraceTree with init + assistant + tool_call + tool_result + result produces correct nesting: init at root, assistant as branch, tool_call nested under assistant, tool_result nested under tool_call
    - Consecutive tool_call + tool_result pairs within an assistant turn all nest under that assistant node
    - thinking entries nest under the preceding/containing assistant turn
    - Duration is calculated as difference between consecutive entries' timestamps (in ms)
    - Entries without timestamps produce durationMs = undefined
    - result entry produces a top-level terminal node with token count from inputTokens + outputTokens
    - stderr and system entries appear as standalone top-level nodes
    - TraceNode.id is unique for each node (use index-based id like "node-0", "node-0-0" for children)
    - TraceNode.label is human-readable: init shows model name, assistant shows truncated text, tool_call shows tool name, result shows "Result"
  </behavior>
  <action>
    Create `ui/src/lib/trace-utils.ts` with:

    1. Define the `TraceNode` interface:
    ```typescript
    export interface TraceNode {
      id: string;
      type: "init" | "prompt" | "tool_call" | "tool_result" | "response" | "thinking" | "result" | "system" | "stderr";
      label: string;
      ts: string;
      durationMs?: number;
      tokenCount?: number;
      children: TraceNode[];
      content: TranscriptEntry;  // Original entry for expanded view
    }
    ```

    2. Implement `buildTraceTree(entries: TranscriptEntry[]): TraceNode[]` that:
       - Maps TranscriptEntry kinds to TraceNode types: assistant->response, user->prompt, init->init, result->result, tool_call->tool_call, tool_result->tool_result, thinking->thinking, stderr->stderr, system/stdout->system
       - Nesting strategy per RESEARCH.md Pattern 1:
         a. `init` entry = top-level root node
         b. Each `assistant` entry starts a new top-level branch
         c. `tool_call` entries nest under the preceding assistant node
         d. `tool_result` entries nest under the preceding tool_call (matched by position/adjacency)
         e. `thinking` entries nest under the containing assistant turn
         f. `result` entry = top-level terminal node with tokenCount = inputTokens + outputTokens
         g. `stderr`, `system`, `stdout` = top-level standalone nodes
       - Duration calculation: for each node, compute ms difference from its timestamp to the next sibling's timestamp. Use `new Date(ts).getTime()` for parsing.
       - Label generation: init -> "Session: {model}", assistant -> first 60 chars of text + "...", tool_call -> "Tool: {name}", tool_result -> "Result" (truncated content), result -> "Run Complete", thinking -> "Thinking...", stderr -> "stderr", system -> "System"
       - IDs: use hierarchical index pattern "node-{i}" for top-level, "node-{i}-{j}" for children, "node-{i}-{j}-{k}" for grandchildren

    Create `ui/src/lib/trace-utils.test.ts` with tests covering all behaviors listed above. Import TranscriptEntry from @paperclipai/adapter-utils. Write tests FIRST (RED), then implement (GREEN).
