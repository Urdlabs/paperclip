<objective>
Build the complete outgoing webhook notification system: DB schema, service layer with Standard Webhooks signing, event-to-webhook dispatcher bridge, retry queue, and REST API routes.

Purpose: Operators need to receive structured notifications when important platform events occur (runs completing/failing, issues created, approvals requested). This is the backend infrastructure that powers all webhook-based alerting.

Output: Two DB tables (webhook_endpoints, webhook_deliveries), webhook service with CRUD + signing + delivery, event dispatcher bridge, retry queue sweep, REST API for endpoint management and delivery log viewing.
</objective>
