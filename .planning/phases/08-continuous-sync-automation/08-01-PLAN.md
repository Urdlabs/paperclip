<objective>
Create the upstream sync GitHub Action workflow and exempt the sync branch from the PR lockfile policy.

Purpose: Automate fork-to-upstream synchronization so the fork never silently drifts. The workflow detects new upstream commits, seeds rerere from Phase 6 conflict resolutions, attempts a merge, and either creates a review-ready PR (clean merge) or a conflict-report issue (conflicts). Lockfile is auto-regenerated. The sync branch is exempted from the existing pr-policy lockfile block so CI does not reject the sync PR.

Output: `.github/workflows/upstream-sync.yml` (new), `.github/workflows/pr-policy.yml` (modified)
</objective>

  <action>
Create `.github/workflows/upstream-sync.yml` with a single job containing sequential steps. Follow the exact patterns from existing workflows (checkout@v4, pnpm/action-setup@v4 with version 9.15.4, setup-node@v4 with node-version 20, cache: pnpm).

**Triggers:** `schedule` with cron `'0 8 * * 1'` (Monday 08:00 UTC) plus `workflow_dispatch` for manual runs.

**Concurrency:** group `upstream-sync`, cancel-in-progress: false (do not cancel ongoing syncs).

**Permissions:** contents: write, pull-requests: write, issues: write.

**Job:** `sync`, runs-on: ubuntu-latest, timeout-minutes: 15.

**Steps (in order):**

1. **Checkout** with fetch-depth: 0 (full history needed for merge, merge-base, rev-list, and rerere seeding).

2. **Setup pnpm** (version 9.15.4, run_install: false).

3. **Setup Node.js** (node-version: 20, cache: pnpm).

4. **Detect upstream changes** (id: `detect`):
   - Add upstream remote as HTTPS: `git remote add upstream https://github.com/paperclipai/paperclip.git 2>/dev/null || true`
   - `git fetch upstream master --quiet`
   - Compare: `UPSTREAM_HEAD=$(git rev-parse upstream/master)` and `MERGE_BASE=$(git merge-base HEAD upstream/master)`
   - If equal, write `up_to_date=true` to `$GITHUB_OUTPUT` and exit 0
   - Otherwise compute `BEHIND_COUNT=$(git rev-list --count HEAD..upstream/master)` and write both `behind_count` and `up_to_date=false` to `$GITHUB_OUTPUT`

5. **Seed rerere from Phase 6 merge** (condition: `if: steps.detect.outputs.up_to_date == 'false'`):
   - `git config rerere.enabled true` and `git config rerere.autoUpdate true`
   - Save position: `ORIG_HEAD=$(git rev-parse HEAD)`
   - Detach to first parent: `git checkout --detach "8c83b0d^1" --quiet`
   - Attempt training merge: `git merge --no-commit --no-ff "8c83b0d^2" >/dev/null 2>&1 || true`
   - Record conflict state: `git rerere`
   - Apply known resolution: `git checkout "8c83b0d" -- . 2>/dev/null`
   - Record resolution: `git rerere`
   - Return: `git checkout --force --quiet "$ORIG_HEAD"`

6. **Attempt upstream merge** (condition: same `up_to_date == 'false'`; id: `merge`):
   - Configure git user: `git config user.name "upstream-sync-bot"` and `git config user.email "upstream-sync-bot@users.noreply.github.com"`
   - Create sync branch: `git checkout -B chore/upstream-sync`
   - Attempt merge: `if git merge upstream/master --no-edit; then echo "merge_clean=true" >> "$GITHUB_OUTPUT"; else echo "merge_clean=false" >> "$GITHUB_OUTPUT"; fi`
   - If merge is clean, the working tree has the merged result ready to proceed.

7. **Regenerate lockfile** (condition: `if: steps.merge.outputs.merge_clean == 'true'`):
   - `rm -f pnpm-lock.yaml`
   - `pnpm install --no-frozen-lockfile`
   - `git add pnpm-lock.yaml`
   - `git commit --amend --no-edit` (amend the merge commit to include lockfile)

8. **Create sync PR** (condition: `if: steps.merge.outputs.merge_clean == 'true'`):
   - Force-push sync branch: `git push --force origin chore/upstream-sync`
   - Generate area-grouped changelog using `git log` with `--name-only` between merge-base and upstream/master. Use a `classify_area()` shell function that maps file paths to areas: `packages/db/*|*/migrations/*` -> DB, `server/*` -> Server, `ui/*` -> UI, `packages/*` -> Packages, `cli/*` -> CLI, `Dockerfile*|docker-compose*|.github/*` -> Infrastructure, `*` -> Other
   - Build PR body with `## Upstream Changes` header, then `### {Area}` sections with commit messages and short hashes
   - Include behind count in PR title: `chore(upstream): sync ${BEHIND_COUNT} commits from upstream`
   - Check for existing PR: `existing=$(gh pr list --head "chore/upstream-sync" --json number --jq '.[0].number')`
   - If no existing PR: `gh pr create --head "chore/upstream-sync" --title "..." --body "$PR_BODY"`
   - If existing: just log that branch was updated via force push

9. **Create conflict issue** (condition: `if: steps.merge.outputs.merge_clean == 'false'`):
   - List conflicting files: `git diff --name-only --diff-filter=U`
   - Group by area using the same `classify_area()` function
   - Build issue body with `## Upstream Sync Conflicts` header, area sections, and file lists
   - Abort the merge: `git merge --abort`
   - Create issue: `gh issue create --title "Upstream sync: merge conflicts detected ($(date -u +%Y-%m-%d))" --body "$ISSUE_BODY" --label "upstream-sync"`
   - Handle label not existing: create it first with `gh label create "upstream-sync" --description "Automated upstream sync" --color "0E8A16" 2>/dev/null || true`

10. **Update sync status** (condition: `if: steps.detect.outputs.up_to_date == 'false'`):
    - Compute current behind count AFTER merge attempt. If merge was clean: behind_count=0. If conflicts: use the original behind_count from detect step.
    - Determine color: 0 -> brightgreen, <10 -> green, <50 -> yellow, else -> red. If conflicts, force red.
    - Determine status text: clean merge -> "synced", conflicts -> "conflicts"
    - Write `.github/sync-status.json` with shields.io endpoint badge schema: `{ "schemaVersion": 1, "label": "upstream sync", "message": "{N} behind | {date}", "color": "{color}" }`
    - For a clean merge where behind_count is 0: message should be `"0 behind | {date}"`
    - For conflicts: message should be `"{N} behind (conflicts) | {date}"`
    - Commit and push the JSON update to master: `git checkout master`, `git add .github/sync-status.json`, `git commit -m "chore: update sync status"`, `git push origin master`
    - IMPORTANT: If merge was clean and PR was created, the JSON update goes to master directly (not the sync branch). This ensures the badge is always up-to-date regardless of PR merge status.

Use `env: GH_TOKEN: ${{ github.token }}` at the job level (matching refresh-lockfile.yml pattern).

IMPORTANT: Use HTTPS URL (not SSH) for the upstream remote — GitHub Actions runners use HTTPS tokens, not SSH keys.
